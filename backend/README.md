# Backend

Hier entsteht die **API des BIW‑Pokal‑Systems**.

## Aufgaben

- Persistenz der Turnierdaten
- Validierung von Zuständen
- Berechnung von Tabellen & KO‑Weiterkommen
- Schnittstelle für n8n & Frontend

## Geplant

- REST API
- SQLite (dev) / Postgres (prod)
- Deterministische Business‑Logik

---

> Keine UI. Keine Automatisierung. Nur Logik.
