# BIW Pokal Tournament System

## Project Overview
BIW Pokal is a tournament management system for managing "Business in Würzburg" football tournaments. The system handles multiple seasons with group phases and knockout rounds.

**GitHub Repository:** https://github.com/cyphrrr/Funcup-Tournament-System

## Tech Stack
- **Backend:** FastAPI (Python) with SQLAlchemy ORM
- **Database:** SQLite (development) / PostgreSQL (production)
- **Frontend:** Vanilla HTML/CSS/JavaScript
- **Deployment:** Docker Compose + Nginx reverse proxy
- **Authentication:** JWT tokens + API key

## Key Features
- **Multi-Season Support:** Track multiple tournament seasons
- **Group Phase:** Teams divided into groups (max 4 teams per group)
- **Knockout Phase:** Semifinals and finals
- **Data Import:** Import historical data from WordPress
- **Admin Interface:** Manage teams, matches, and results
- **Public Interface:** View standings, matches, and statistics

## Common Issues & Solutions

### Issue 1: 5-Team Season Groups
**Problem:** Creating a season with 5 teams causes 2-group split (ceil(5/4)=2)
**Solution:** Create season with 4 participants first, then add 5th team individually via API

### Issue 2: Manual KO Match Setup
**Problem:** Cannot manually assign teams to KO matches for historical imports
**Solution:** Extended `KOMatchUpdate` schema to include `home_team_id` and `away_team_id`

### Issue 3: Frontend "Backend nicht erreichbar" in Production
**Problem:** Frontend using `const API = 'http://localhost:8000'` doesn't work for external access
**Solution:** Use `const API = ''` (relative URLs) since nginx is configured as reverse proxy

### Issue 4: Docker Healthcheck Failure
**Problem:** Healthcheck using `curl` command fails (curl not in container)
**Solution:** Use Python urllib instead:
```yaml
test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"]
```

### Issue 5: Wrong Database Location
**Problem:** biw.db placed in root funcup/ directory instead of funcup/backend/
**Solution:** Move biw.db to funcup/backend/ directory for proper Docker volume mounting

## Development Notes

### Adding New Features
1. Modify backend code (api.py, schemas.py, models.py)
2. Update frontend HTML files if needed
3. Test locally with `docker compose up -d`
4. Commit and push to GitHub
5. Pull on production server and restart services

### Database Migrations
- Currently no migration system (Alembic not configured)
- Schema changes require manual updates or database recreation
- For production: backup biw.db before major changes

### Frontend Development
- All HTML files use inline CSS and JavaScript
- API calls use relative URLs (`/api/...`)
- Authentication token stored in localStorage
- Real-time updates via periodic API polling (no WebSocket)

## Useful API Endpoints

```
GET  /api/seasons                    # List all seasons
POST /api/seasons                    # Create season
GET  /api/seasons/{id}               # Season details
POST /api/seasons/{id}/teams         # Add team to season
GET  /api/seasons/{id}/standings     # Group standings
GET  /api/matches                    # List all matches
PATCH /api/matches/{id}              # Update match result
GET  /api/ko-matches                 # List KO matches
PATCH /api/ko-matches/{id}           # Update KO match
POST /api/login                      # Get JWT token
```
